# 와니온봇 (WANION Discord Bot) — 설계 & 세팅 정리

> 작성: 2026-07-22 · 수진 · 상태: 설계 v1 (한길련 봇 검증 구조 계승)
> 대표님이 하실 세팅은 `WANION_대표님_세팅_체크리스트.md` 4번 섹션에 단계별로 있습니다. 이 문서는 봇의 전체 그림.

## 1. 아키텍처 — 한길련 봇 방식 그대로 (서버비 0원)

```
디스코드 서버 ──(슬래시 명령/버튼 클릭)──▶ Discord가 우리 URL 호출
                                              │
                              Cloud Functions (HTTP Interactions)
                              · Public Key로 서명 검증
                              · 명령 처리 → Firestore 읽기/쓰기
                              · 응답/카드 게시
                                              │
Firestore 변경(신청·픽스·포인트) ──(트리거)──▶ 카드보드 갱신 · 알림 발송
```

- **상주 프로세스 없음**: 봇이 24시간 켜져 있는 서버가 아니라, 디스코드가 부를 때만 Functions가 깨어나는 구조. 한길련 봇과 동일 — 비용 0원대, 관리 부담 최소
- **봇 1개가 N개 서버 서비스**: 한길련 디코 + TeamSAD 디코 + (미래) 등록 길드 디코들. 서버↔커뮤니티 연결은 Firestore에 저장 (kgusystem `cardChannels` 구조 계승·확장)
- 봇 토큰/시크릿은 Firebase Functions Secrets (체크리스트 8번)

## 2. 기능 — 계승 + 신규

### 계승 (한길련 봇에서 이식)
| 기능 | 설명 |
|---|---|
| 카드보드 | 채널에 다가오는 공대 카드 자동 게시·갱신 (모집 현황 실시간) |
| /일정 | 다가오는 레이드 목록 임베드 |
| /신청 · /취소 · /변경 | 디코에서 바로 공대 신청 (웹과 동일 자격 검증 — minIlvl 포함, P0에서 수정 완료) |
| /내신청 | 내 신청 현황 |
| /연동 | 웹에서 발급한 코드로 디코 계정 ↔ WANION 계정 연결 |
| 포럼 자동 생성 | 길드별 모집 포럼 프로비저닝 |

### 신규 (WANION 전용)
| 기능 | 설명 |
|---|---|
| **픽스 알림** | 공대장이 픽스하면 해당 멤버 멘션 "출발 확정 — 20:00 집결" |
| **포인트 알림** | 일정 종료 후 "+100P 지급 (이번 주 2/3)" DM 또는 채널 알림 |
| **노쇼 신고 접수** | 공대장이 웹/디코에서 신고 → 관리자 채널에 심사 카드 (승인/기각 버튼) |
| **제명 제안 알림** | BNet 로스터 감시가 전원 탈퇴 감지 → 길드마스터에게 [제명/유지/보류] 카드 |
| **쪽지 DM 브릿지** | WANION 새 쪽지 도착 시 디코 DM으로 알림 |
| **프로필 카드** | /프로필 → 누적P·계급·프로그레스 카드 임베드 (꺼드럭 장치) |
| **역할 동기화(추후)** | WANION 멤버십 → 디코 역할 자동 부여 (길드원/공대원/계급) |

## 3. 명령 체계 (v1 안)

```
/일정 [전체|내공대]      /신청 <공대> <캐릭터>     /취소 <공대>
/내신청                  /프로필 [@유저]           /연동 <코드>
/픽스 <공대>            ← 공대장 전용 (웹과 동일 동작)
/신고 노쇼 <공대> <@유저> <사유>  ← 공대장 전용, 관리자 심사로 전달
```

## 4. 세팅 순서 (누가 뭘 하는지)

| 순서 | 담당 | 작업 |
|---|---|---|
| 1 | 대표님 | 디스코드 앱 "WANION" 생성 + 봇 토큰 발급 + 서버 초대 (체크리스트 4-A~4-D) |
| 2 | 대표님 | `firebase functions:secrets:set DISCORD_BOT_TOKEN` 등 시크릿 등록 (체크리스트 8) |
| 3 | 수진 | Functions에 인터랙션 핸들러 배포 → URL 전달 |
| 4 | 대표님 | Interactions Endpoint URL 붙여넣기 (체크리스트 4-E) |
| 5 | 수진 | 슬래시 명령 등록 스크립트 실행, 카드보드 채널 연결 |
| 6 | 공동 | 한길련 디코에서 /일정 테스트 → TeamSAD 디코 확장 |

## 5. 한길련 기존 봇과의 관계

- 이행기: 한길련 디코에는 KWGU봇과 와니온봇이 **한동안 공존** → 와니온봇 안정 확인 후 KWGU봇 은퇴
- kgusystem `functions/discord-bot.js`(1,500줄)가 이식 원본 — 서명 검증·카드보드·명령 라우팅 구조를 WANION 스키마(스코프·memberships)에 맞게 리팩토링하며 가져옴
- 중복 상수(클래스·정원)는 이번에 웹과 단일 소스로 통합 (한길련 감사에서 나온 3중 복제 문제 해소)
