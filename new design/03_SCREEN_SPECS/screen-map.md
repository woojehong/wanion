# 화면별 설계 사양

## 공통 헤더

데스크톱:

- 좌측: WANION 워드마크
- 중앙: 파티 찾기 / 공략 / 게시판 / 길드 / 공격대
- 우측: 파티 개설, 알림, 프로필
- `RUN BY STUDIO HOOJE`는 헤더가 아니라 랜딩·푸터 우선

모바일:

- 상단 48–52px
- 워드마크, 알림, 메뉴
- 주요 메뉴는 드로어 또는 하단 내비게이션
- 가로로 메뉴 전체를 흘리는 방식은 임시안으로만 허용

## 랜딩 `/`

우선순위:

1. 한 문장 가치: 길드·공격대·연합의 일정을 한곳에서 운영
2. 지금 모집 중인 파티 보기
3. 주간 운영 주기
4. 길드/공격대 공개 프로필
5. 핵심 기능: 시너지·벤치·스왑·배치
6. WANI 단일 사용

히어로에 거대한 AI 판타지 키아트를 두지 않는다. 제품 화면과 정보 구조가 주인공이어야 한다.

## 파티 찾기 `/board`

상단:

- 제목, 모집 중 수, 이번 주 일정 수
- `파티 개설`
- 상시 주간 스트립

필터:

- 주최: 전체/연합/길드/공격대/개인
- 난이도
- 역할 빈자리
- 날짜
- 손님파티 여부
- 보드/달력 전환

리스트 행:

- 상태, 난이도, 제목
- 주최와 공대장
- 날짜·시간·최소 아이템 레벨
- 탱/힐/딜 현재/정원과 슬롯
- 대기 인원
- 마감까지 남은 시간

## 달력

- 월간 기본, 주간 스트립 유지
- 날짜 셀 안 이벤트: 시간, 제목, 현재/정원
- 동일 시간 충돌은 겹치지 않고 세로 정렬
- 하루 3개 초과: 우선 3개 + `+N개 더보기`
- 선택 시 우측 프리뷰 또는 하단 시트
- 모바일은 월간 밀도가 부족하면 3일/주간 어젠다로 전환

## 레이드 상세 `/raid/:id`

정보 순서:

1. 제목·주최·날짜·난이도·상태
2. 내 신청 상태와 주요 CTA
3. 역할 정원·현재 인원·대기
4. 로스터
5. 시너지
6. 대기·벤치·스왑
7. 운영자 도구
8. 기록·취소·로그

공대장 도구:

- 일반 신청자 화면과 분리
- 데스크톱 우측 운영 패널 또는 별도 탭
- 모바일에서는 `운영 도구` 진입 후 별도 화면

### 레이드 생성·편집

- 진입: 파티 보드의 `파티 개설`, 상세의 `일정 편집`
- 단계: 기본 정보 → 모집·정원·수락 방식 → 운영 옵션·확인
- 상태: 신규, 기존 일정 편집, 과거 일정 복사, 손님파티 전환 경고
- 삭제·아카이브와 일반 닫기는 명확히 분리

### 신청·취소·승인

- 신청: 캐릭터·역할·특성·메모 확인 후 제출
- 대기만 가능한 사용자는 사유와 대기 예상 상태 표시
- 취소: 사유 입력, 영향 설명, 재신청 가능 여부
- 관리자: 승인·거절·대기 전환·벤치 이동
- 거절과 강제 취소는 확인 모달 및 기록 생성

## 내 일정 `/me`

데스크톱 2패널:

- 좌측: 날짜순 일정 목록, 상태와 충돌
- 우측: 선택 일정 상세, 내 역할, 신청 메모, 취소/변경

상단 요약:

- 이번 주 확정
- 대기
- 운영 중
- 충돌

NION은 지난 일정 회고나 미처리 기록이 있을 때만 제한적으로 사용한다.

## 길드 `/guild/:id`

외부인:

- 로고, 소개, 모집 상태, 프로그레스, 가입 신청

소속원:

- 게시판이 기본
- 소개, 일정, 구성원, 공략

관리자:

- 관리 페이지 진입은 권한자에게만 표시
- 로고 업로드, 카테고리, Discord, 가입 신청

## 공격대 `/team/:id`

- 공개 소개·프로그레스·WCL 리포트
- 소속원 게시판과 일정
- 정규 로스터
- 공대장/관리자 전용 설정
- WCL 항목별 공개 범위

### 공격대 관리

- 정규 로스터 편집
- 운영진과 권한
- WCL 길드 연결·갱신·공개 범위
- 손님 유형 프리셋
- Discord 카드 채널
- 일반 소속원에게 관리 진입 자체를 노출하지 않음

## 공략 `/guides`

- 던전→네임드→글 목록
- BEST 3과 일반 목록의 위계 분리
- 도움/비도움 투표는 숫자보다 의미를 먼저
- 공개 범위와 난이도 필터
- 검색 유입을 고려해 본문 가독성 우선

## 게시판 `/community`

- 공지/자유/구인구직/거래
- 카테고리 관리형
- 목록은 전통적인 게시판 가독성을 유지
- 카드식 SNS 피드로 만들지 않는다.

## 관리자 `/admin`

- 운영자용 밀도 높은 테이블
- 일반 제품 화면보다 캐릭터와 마케팅 요소를 더 줄인다.
- 필터, 일괄 작업, 상태, 변경 이력 우선
- 위험 동작은 영향 범위 확인

## 주요 상태 화면

- 로딩: 스켈레톤, 레이아웃 이동 최소
- 빈 상태: 설명 + 다음 행동. 필요할 때 WANI 사용
- 오류: 기술 코드보다 해결 방법
- 권한 없음: 필요한 권한과 문의 대상
- 미연동: BNet/Discord 연동 이유와 단계
- 충돌: 어느 일정과 겹치는지 정확히 표시

## 부가 흐름

### 길드 가입 신청

- BNet 검증 상태
- 해당 길드 캐릭터 확인
- 대표 캐릭터와 신청 메시지
- 관리자 승인·유지·보류·거절

### 알림

- 신청 상태, 일정 변경, 승격·강등, 스왑, 쪽지, 연결 오류
- 읽음/안읽음, 관련 화면 이동, 일괄 읽음

### 프로필 연결 관리

- Google, Discord, Battle.net, WCL 상태
- 필수 연결과 선택 연결을 구분
- 토큰·자격증명 자체를 UI나 로그에 표시하지 않음
