# WANION Design System vNext

## 1. 디자인 방향

권고 조합은 `RAID DESK`의 정보 구조 + `WANE INDEX`의 주간 상태 언어다.

- RAID DESK: 로스터 바, 기준선, 슬롯, 운영 콘솔
- WANE INDEX: 모집→구성→확정→출발→기록의 주기
- SWITCHBOARD: 상태 탭은 보조적으로만 사용

목표는 “어두운 게임 사이트”가 아니라 “레이드 운영을 위해 설계된 현대적 편집형 제품”이다.

## 2. 색상

```css
--ink-950: #09090D;   /* 전체 배경 */
--ink-900: #111116;   /* 주요 패널 */
--ink-850: #17171D;   /* 상승 패널 */
--line:    #292832;   /* 기본 경계 */
--text:    #F1F0F5;   /* 제목 */
--sub:     #A09DA9;   /* 보조 본문 */
--mute:    #6E6B77;   /* 비활성 */
--violet:  #8A70FF;   /* 핵심 액션·활성 */
--violet-hi:#B9A8FF;  /* 활성 텍스트 */
--violet-deep:#5B45E0;
--paper:   #F7F4ED;   /* 캐릭터·편집형 밝은 면 */
```

역할색은 기존 의미를 유지하되 작은 슬롯·텍스트에만 제한한다. 넓은 카드 배경으로 쓰지 않는다.

바이올렛 사용:

- 주요 CTA
- 현재 선택
- 핵심 수치
- 월상 진행 단계
- WANI/NION의 작은 탭

한 화면 바이올렛 면적은 대략 12–18%를 넘기지 않는다.

## 3. 타이포그래피

- 국문·본문: Pretendard Variable
- 숫자·시간·상태 코드: 시스템 모노스페이스
- Studio Hooje 연결용 영문 에디토리얼: Marcellus는 마케팅 면에서 제한적으로 사용 가능
- 본문 최소 14px, 모바일 보조 텍스트 최소 12px
- 과도한 대문자 영문 라벨과 자간은 섹션 보조 정보에만 사용

## 4. 공간과 그리드

- 기본 단위: 4px
- 주요 간격: 8 / 12 / 16 / 24 / 32 / 48 / 64
- 콘텐츠 최대 폭: 1200–1280px
- 데스크톱 12열, 태블릿 8열, 모바일 4열
- 카드 간격보다 섹션 간격을 크게 하여 정보 덩어리를 구분

## 5. 모서리와 경계

- 버튼 6–8px
- 카드 8px
- 작은 칩 999px 사용 가능하나 모든 UI를 알약 형태로 만들지 않는다.
- 기본 경계 1px, 선택 상태는 색·배경·텍스트를 함께 변경
- 그림자보다 면과 경계 차이를 사용

## 6. 버튼

### Primary

- 바이올렛 단색, 검은 텍스트 또는 충분한 대비의 흰 텍스트
- 한 화면에 가장 중요한 행동 1개
- 글로우 금지

### Secondary

- 투명 또는 상승 패널 배경 + 경계
- Primary와 같은 높이

### Tertiary

- 텍스트 버튼. 삭제·취소·부가 기능

### Destructive

- 항상 결과 설명 또는 확인 단계 포함
- 빨강은 파괴적 행동과 오류에만 제한

## 7. 입력

- 라벨을 placeholder로 대체하지 않는다.
- 기본 높이 40–44px
- 도움말과 오류문을 동일 위치에 표시
- 복잡한 레이드 생성은 한 장의 거대 모달이 아니라 3단계 또는 논리 섹션으로 구분

## 8. 카드와 행

- 탐색 보드는 카드보다 “정보 행”에 가깝게 설계
- 좌측: 난이도·제목·주최
- 중앙: 날짜·시간·조건
- 하단/우측: 역할 슬롯·마감·신청
- 동일 정보를 배지로 반복하지 않는다.

## 9. 월상 상태 인덱스

| 상태 | 기호 | 의미 |
|---|---|---|
| 모집 | ○ | 파티 공개, 신청 시작 |
| 구성 | ◔ | 신청이 쌓이는 중 |
| 확정 | ● | 로스터 픽스 |
| 출발 | ◕ | 체크인·교체 처리 |
| 기록 | ◑ | 종료·회고·다음 주 준비 |

기호 옆에 상태 텍스트를 반드시 함께 쓴다. 오컬트·점성술 장식으로 사용하지 않는다.

## 10. 캐릭터 사용

- 활성 원본: `06_BRAND_CHARACTERS/WANI_canonical.png`, `NION_canonical.png`
- WANI: 랜딩의 제품 안내, 온보딩, 모집 빈 상태
- NION: Discord, 기록·회고, 오류가 아닌 주의·미완료 상태
- 한 화면에서 캐릭터는 최대 1회
- 데이터가 많은 운영 화면의 카드마다 반복 삽입 금지
- 표정 스티커·밈 캐릭터처럼 남용 금지

## 11. 크레딧

- 표기: `RUN BY STUDIO HOOJE`
- WANION 워드마크 크기의 28–32%
- 헤더에서는 상시 노출하지 않아도 되며 랜딩·푸터·About에 제한
- Studio Hooje의 H 마크를 WANION 심벌로 사용하지 않는다.

## 12. 금지 시각 문법

- 보라색 그라디언트
- 외곽 발광과 네온 테두리
- 유리 질감·과도한 블러
- 포털, 우주 배경, 별자리, 신경망, 회로, 육각형
- 의미 없는 3D 구체와 플로팅 카드
- AI가 자주 만드는 과도한 대문자 마이크로 카피
- WoW 공식 아트·로고·진영 문양과 혼동되는 요소
