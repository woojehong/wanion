# 캐릭터 생성 프롬프트 기록

도구 모드: OpenAI 내장 이미지 생성, 신규 래스터 이미지 생성. 기존 이미지 편집이나 외부 레퍼런스 복제는 사용하지 않았다.

## 공통 프롬프트

WANION은 공격대 모집·명단·일정·교체 상태를 다루는 프로페셔널 운영 플랫폼이다. 따뜻한 오프화이트 배경에 캐릭터의 전신 4포즈와 Discord 원형 크롭을 포함한 편집 디자인형 캐릭터 시트를 생성한다. 평면 색, 또렷한 외곽선, 작은 크기에서도 읽히는 실루엣을 사용한다. 그라디언트·발광·3D·텍스트·게임 프랜차이즈 상징·AI 클리셰는 금지한다. 팔레트는 `#09090D`, `#8A70FF`, `#B9A8FF`, `#F7F4ED`로 제한한다.

## 방향별 최종 프롬프트 세트

1. **MOA / RAID DESK**  
   검은 담비 운영관. 중립 자세, 클립보드로 명단 확인, 대기자를 확정 칸으로 옮기는 동작, 원형 아바타. 차분하고 빈틈없는 표정과 작은 이어피스.

2. **LILI / THE BATON**  
   아이보리 페럿 인계 담당자. 중립 자세, 작은 바통 케이스 전달, 다음 담당자를 가리키는 동작, 원형 아바타. 다정하지만 들뜨지 않은 표정.

3. **TEMPO / RAID SCORE**  
   검은 학 큐 마스터. 중립 자세, 시간 신호 확인, 순서를 지휘하는 동작, 원형 아바타. 긴 실루엣과 절제된 큐 제스처.

4. **PICO / SWITCHBOARD — v2**  
   검은 딱따구리 상태 콘솔 운영자. 중립 자세, 작은 사각 상태 탭 조작, 열린 프레임 대시보드 표시, 원형 아바타. 머리 깃을 포함한 모든 요소에 빨강·주황·노랑·파랑·초록을 절대 사용하지 않고, 근흑색과 바이올렛만 사용.

5. **BAN / COMMON FLAG**  
   검은 까치 커뮤니티 코디네이터. 중립 자세, 비대칭 공동 깃발 정리, 여러 파티를 한곳으로 부르는 동작, 원형 아바타. 친근하되 마스코트 상품처럼 유아적으로 보이지 않는 표정.

6. **VANE / WANE INDEX — archived v1**  
   `wanion`의 ‘기우는 달’ 어원을 주간 운영 주기로 바꾼 검은 달나방. 중립 자세, 4단계 월상 인덱스 표시, 다음 운영 주기 안내, 원형 아바타. 날개의 여백이 기우는 초승달을 암시하되 점성술·오컬트·별·마법·고딕 메탈 표현은 사용하지 않는다.

7. **NION / WANE INDEX — v2**  
   정사각형 2×2 편집 시트. 자연형에 가까운 가늘고 성숙한 검은 달나방으로, 작은 무광 눈과 얇은 선을 사용한다. 월상은 배경 장식이 아니라 로스터 카드의 4단계 상태 인덱스로만 표시한다. SD 비례·큰 눈·광택·그라디언트·오컬트·고딕 표현과 문자 로고를 금지한다.

8. **ONI / CURSE KEEPER**  
   5개 브랜드 방향과 별개의 여섯 번째 세계관 캐릭터. 자연형에 가까운 검은 까마귀가 취소된 로스터 쪽지를 회수해 검은 아카이브 봉투에 넣는다. 저주를 걸지 않고 반복되는 실패 기록을 보관하는 역할이다. 2×2 정사각 시트, 절제된 표정, 무광 평면색. 해골·룬·마법·오컬트·점성술·고딕·메탈 밴드·공포·무기·문자 로고를 금지한다.

## 생성 결과와 최종 자산의 차이

이미지 생성 프롬프트에서는 평면색을 요구했지만 NION v2와 ONI의 일부 보라색 요소에 미세한 명암 변화가 남았다. 따라서 이 파일들은 조형·비례·역할을 비교하는 콘셉트 시트이며, ‘완성된 무광 평면 자산’으로 간주하지 않는다. 선택 후 벡터 재작도 단계에서 모든 그라디언트와 광택을 제거해야 한다.

## 한계

- 생성 이미지는 방향 선택을 위한 콘셉트 시안이며 최종 로고 원본이나 상표 출원용 도안이 아니다.
- 이미지 안의 작은 소품 표식은 최종 선택 후 수작업으로 정리해야 한다.
- 최종 캐릭터는 SVG 재작도, 24/32px 축소 검증, Discord 원형 크롭 검증이 필요하다.
